== docker compose up -d (to start service ) and docker compose down( to stop service) and check running containers (docker ps)

# ☀️ Morning: Start working (ALWAYS use this)
docker compose up -d   # Safe - creates OR starts containers

# 💻 During the day: Quick pause/resume
docker compose stop    # Fast pause
docker compose start   # Fast resume

# 🌙 Evening: Clean shutdown
docker compose down    # Removes containers, KEEPS data

# 🧹 Rare: Complete reset (only when you want to wipe everything)
docker compose down -v # Removes containers AND data

all scripts ---
docker ps (to check running containers)
Command	 		What it does					Container State		Data
docker compose up -d	Creates and starts containers in background	✅ Running		✅ Kept
docker compose start	Starts existing stopped containers		✅ Running		✅ Kept
docker compose stop	Gracefully stops running containers		⏸️ Stopped		✅ Kept
docker compose down	Stops AND removes containers			❌ Removed		✅ Kept
docker compose down -v	Stops, removes containers AND deletes volumes	❌ Removed		❌ Deleted
